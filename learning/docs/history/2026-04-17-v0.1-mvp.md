# 2026-04-17 v0.1 — MVP 構築から完全独立化まで

学習ドリルアプリの誕生から初版コミットまでのジャーニー。ADR は個別判断を凍結する場所なので、ここでは**なぜ / いつ / どの順で**決まっていったかの文脈を残す。

## 発端 (2026-04-16)

ユーザー (個人投資家、SE 素養あり、体系的な金融教育経験なし) が、Master Sensei のレポートで頻出する用語の理解に苦労していると申告。Claude から session 内で 17 用語の診断を実施:

- A (自分の言葉で 2 文説明可): 2 用語 (ADR, SL/TP+OCO)
- B (直感あり、tightening 必要): 5 用語 (ETF, 平均回帰, 3x レバ, regime_assessments, Decay)
- C (未知): 11 用語 (VIX, MAP, EPS, コンタンゴ, σ/SMA, YC, NII, guidance, BE SL, K-XXX, Section 232)

**重要な観測**: 診断でユーザーは C=11 を honest に選んだ。Dunning-Kruger バイアスが低いと確認でき、self-grading 方式が機能する前提が成立。これが ADR-003 (grading 方式) の決定的根拠になった。

## 要求定義

ユーザーからの要件:
- セッション独立 (Claude に毎回頼らない)
- 解説 + クイズの pair が continuable (続けやすい)
- 公文式的ドリル (反復・漸進・即フィードバック)
- Python CLI で OK (SE 素養あり)
- Claude 側は定期レビューでカリキュラム調整

## リサーチフェーズ

学習科学の 3 領域をリサーチ:

1. **FSRS vs SM-2 vs Leitner**: 50-200 問規模では Leitner で十分 (Expertium benchmark)
2. **Free recall vs MCQ**: Free recall が retention / transfer で優位 (Instructional Science 2020)
3. **Feedback 効果**: Any feedback > no feedback (UCSD retrieval practice)

これを踏まえて 3 ADR を同時に起草: Leitner / Markdown bank / Self-rating。
