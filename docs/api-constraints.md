# API制約と運用ガイド

## Tiingo API

公式: https://www.tiingo.com/about/pricing

| 項目 | Free tier | 備考 |
|------|----------|------|
| リクエスト/時間 | 50 | コード側で2秒間隔のスロットリングを実装済み |
| リクエスト/日 | 1,000 | |
| Unique Symbols/月 | 500 | 現在6シンボル使用（1.2%） |
| IEXデータポイント/リクエスト | 10,000 | 2026-03-26実測。旧ドキュメント(2026-02-07)では2,000だったが拡大を確認 |
| EODデータポイント/リクエスト | 制限なし | 全期間1リクエスト |

### 本システムの消費量

| 操作 | リクエスト数 |
|------|------------|
| 日足更新（6シンボル） | 6回 |
| 5分足更新（5シンボル、VIXY除く） | 5回 |
| 合計/実行 | 11回 |

50回/時間の制限に対して十分余裕あり。

### 実装上の制約対応

- `TiingoFetcher._throttle()`: リクエスト間隔2秒を確保
- IEX 10,000ポイント到達時にwarningログ出力
- キャッシュ差分更新により不要なAPIコールを回避

## FRED API

公式: https://fred.stlouisfed.org/docs/api/api_key.html

| 項目 | 制限 |
|------|------|
| リクエスト/分 | 120 |
| 日次制限 | なし（実質） |

9シリーズ × 1リクエスト = 9回/実行。制限に対して十分余裕あり。
