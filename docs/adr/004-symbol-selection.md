# ADR-004: 銘柄選定基準とシンボル追加ポリシー

Status: accepted
Date: 2026-03-26

## Context

ADR-002でTiingo 6シンボルを採用したが、選定根拠が「既存backtestプロジェクトからの引き継ぎ」であり、Master Sensei独自の判断基準がなかった。

また、5分足データはTiingo IEXの制約（1リクエスト最大10,000ポイント ≈ 128営業日）により、蓄積開始が遅れるほど過去データを失う。将来取引対象になりうるシンボルは、取引開始前からデータ蓄積を始めるべき。

## 1. 銘柄選定基準

### 必須条件（すべて満たす）

| 条件 | 理由 | 検証方法 |
|------|------|---------|
| 3xレバレッジETF（ETNは除外） | ETNは発行体の信用リスクあり（BMO等） | 商品種別を確認 |
| 平均日次出来高 > 500K | 短期トレードのスリッページ回避 | Tiingo日足Volumeの20日平均 |
| Tiingo IEX対応 | 5分足データ取得に必要 | API疎通確認 |
| direction.mdのScope内 | 米国レバレッジETF | — |

### 優先基準（スコアリング）

| 基準 | 重み | 理由 |
|------|------|------|
| 出来高（流動性） | 高 | スリッページ・約定力に直結 |
| 既存シンボルとのセクター分散 | 高 | 同一セクター偏重は機会損失 |
| 既存シンボルとの相関の低さ | 中 | 高相関なら追加の情報価値が低い |
| Bear側対応シンボルの存在 | 低 | ヘッジ・逆張り分析に有用 |

## 3. Decision

> **現行6シンボルを維持しつつ、以下3シンボルの5分足蓄積を即時開始する。**

### 即時追加（5分足蓄積開始）

| Symbol | 理由 |
|--------|------|
| SQQQ | Nasdaq-100 Bear。TQQQの逆。Tier 1流動性。Bull/Bear両面分析に必須 |
| TNA | Russell 2000 Bull。既存シンボルにない小型株セクター。Tier 1流動性 |
| TZA | Russell 2000 Bear。TNAの逆。Tier 1流動性 |

### 追加しないもの（理由付き）

| Symbol | 見送り理由 | 再検討トリガー |
|--------|-----------|---------------|
| SPXS | SPXLの逆だが、SPXLはS&P500でTQQQと高相関。セクター分散の寄与が小さい | SPXLをアクティブにトレードし始めた場合 |
| UPRO/SPXU | SPXLと同一セクター（S&P500）。ProSharesとDirexionの差はあるが冗長 | SPXLとの挙動差を検証する必要が出た場合 |
| UDOW/SDOW | Dow Jonesは30銘柄でS&P500と高相関 | Dow固有の動きが分析対象になった場合 |
| FAS/FAZ | 金融セクター。流動性がTier 2下位 | 金融セクター固有のトレード戦略を検討する場合 |
| FNGU/BULZ | ETN（Bank of Montreal信用リスク）。選定基準の「ETF限定」に抵触 | ETNリスクを許容する方針変更時 |
| LABU/LABD | バイオは出来高がBear側に偏り、Bull側がTier 3 | LABUの出来高が回復した場合 |

### 日足のみ追加（5分足は不要）

| Symbol | 理由 |
|--------|------|
| TECS | TECLのBear側。日足での逆相関分析に有用。5分足は出来高が低く不要 |

## 4. 5分足蓄積の優先度とAPI制約

### 制約

- Tiingo IEX: 1リクエストで最大10,000ポイント（≈ 128営業日の5分足）
- 過去データは蓄積でしか増えない。開始が遅れた分だけ失われる
- Free tier: 50リクエスト/時間

### 蓄積コスト

| 現在 | 追加後 | 増分 |
|------|-------|------|
| 5シンボル × 5分足 = 5 req | 8シンボル × 5分足 = 8 req | +3 req/実行 |
| 6シンボル × 日足 = 6 req | 8シンボル × 日足 = 8 req | +2 req/実行 |
| 合計: 11 req/実行 | 合計: 16 req/実行 | 50 req/時間の制限内 |

## 5. シンボル追加のトリガー条件

以下のいずれかに該当した場合、新規シンボルの追加を検討する:

| トリガー | アクション |
|---------|----------|
| 新しいセクターでのトレード戦略を検討開始 | 該当セクターのTier 1-2シンボルを候補に |
| 既存シンボルの出来高が500K未満に低下 | 代替シンボルへの入れ替えを検討 |
| Bear側の分析が必要になった | 対応するBearシンボルを追加 |
| ユーザーが特定シンボルのトレードを開始 | 即座に5分足蓄積開始 |
| 四半期レビュー | ユニバース全体の出来高・流動性を再確認 |

## 6. シンボル除外のトリガー条件

| トリガー | アクション |
|---------|----------|
| 出来高が6ヶ月連続で500K未満 | 5分足蓄積停止を検討 |
| レバレッジ倍率が変更された（3x→2x等） | 選定基準を再評価 |
| ETFがETNに転換、または上場廃止 | 即除外 |
| 6ヶ月間一度もトレード分析に使用されなかった | 日足のみに降格を検討 |

## Consequences

- src/tiingo_client.py の TRADING_SYMBOLS に SQQQ, TNA, TZA を追加
- REFERENCE_SYMBOLS に TECS を追加（日足のみ）
- update_data.py で新シンボルの蓄積を即時開始
- ADR-002の「対象シンボル」セクションはこのADRが上書き
- 四半期レビュー時にユニバーステーブルの出来高を再確認
