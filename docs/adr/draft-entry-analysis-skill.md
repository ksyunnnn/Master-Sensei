# Draft: /entry-analysis スキル

Status: draft（未確定。session 3の議論メモ）
Date: 2026-04-01

## 背景

Session 3でTrade #1（SOXL）を実行した際、以下の問題が発生した：

1. **entry_reasoningの記録タイミング** — 利確後にMaster Senseiが会話を要約して記録した。エントリー時点ではなく事後記録のため、後知恵バイアスのリスクがある
2. **記録者の問題** — ユーザーの意思決定をMaster Senseiが要約している。ユーザー自身の言葉ではない
3. **再現性の欠如** — 会話を振り返って手動で要約。フォーマットが標準化されていない

## Session 3で議論された解決案

`/entry-analysis` スキルを作り、分析→約定→記録を1フローで完結させる。

### 想定フロー

1. ユーザーが「〇〇に入りたい」と言う
2. MAP分析を実行（regime, VIX, Brent, テクニカル, イベント）
3. 銘柄比較・確率計算（Polygon 5年データ、レジームフィルタ相関）
4. エントリー条件の提示（松竹梅のパターン、OCO設定値）
5. ユーザーが約定を確認
6. → `add_trade()` を自動実行（分析結果をentry_reasoningに記録）

### 解決する問題

- **後知恵バイアスの排除** — 分析結果がエントリー時点で自動記録される
- **フォーマット標準化** — MAP分析の各スコア、シナリオ確率、銘柄選定根拠が毎回同じ構造で記録される
- **スナップショットの自動取得** — regime_at_entry, vix_at_entry, brent_at_entryをParquetから自動取得して記録

### Session 3で実際に行った分析の流れ（参考）

1. escalation vs de-escalation のシナリオ確率評価（55%/30%/15%）
2. 全10銘柄の日次パフォーマンス・流動性・ギャップリスク比較
3. Polygon 5年データでVIXY/USO相関をレジーム別に計算
4. 危機タイプ別（関税ショック vs イラン戦争）の相関分離
5. +10%到達確率と-10%到達確率のEdge計算
6. 松竹梅のポジション構成提案
7. 始値別のIFD-OCOパターン提示
8. 開場後のリアルタイム価格確認 → 成行約定 → OCO設定

### 未決定事項

- スキルのスコープ（分析のみ？約定確認+記録まで含む？）
- MAP分析の標準化度合い（毎回同じ4要素？カスタマイズ可能？）
- discipline_scoreの扱い（Session 3でEdgewonk的スコアリングは主観バイアスが高いと評価。客観フィールドでカバー済みなので不要かもしれない）
