# バイアス監査チェックリスト

重要な判断を下す前に実施する。2段階で構成。

## いつ使うか

以下のいずれかに該当する判断:
- 研究の方向性を変える（目標変更、手法変更）
- データやフィルタの設計を変える
- 「これで十分」と打ち止めにする
- 計画を縮小・拡大する

## Step 1: Premortem Analysis

出典: Klein, G. (2007). "Performing a Project Premortem." Harvard Business Review, 85(9), 18-19.

### 手順

1. 「この判断は大失敗した」と未来完了形で宣言する
2. 失敗原因を**5つ以上**列挙する（少なくとも1つは「自分のバイアス」に起因するもの）
3. 各原因の発生可能性（高/中/低）と影響度（大/中/小）を評価する
4. 発生可能性「高」かつ影響度「大」の原因に対して予防策を設計する

### テンプレート

```
判断: [判断内容を1文で]

失敗原因:
1. [原因] — 可能性: 高/中/低, 影響: 大/中/小
2. [原因] — 可能性: 高/中/低, 影響: 大/中/小
3. [原因] — 可能性: 高/中/低, 影響: 大/中/小
4. [原因] — 可能性: 高/中/低, 影響: 大/中/小
5. [バイアス起因の原因] — 可能性: 高/中/低, 影響: 大/中/小

要対策（可能性高×影響大）:
- [原因X]: [予防策]
```

## Step 2: Kahneman 12質問

出典: Kahneman, D., Lovallo, D., & Sibony, O. (2011). "Before You Make That Big Decision." Harvard Business Review, 89(6), 50-60.

各質問に ✅（問題なし）/ ⚠️（疑いあり）/ ❌（問題あり）で判定する。

### 12質問

| # | 質問 | 検出バイアス | 判定 |
|---|------|------------|------|
| 1 | 利己的動機（作業量削減、成果誇張等）の疑いは？ | 利己バイアス | |
| 2 | 自分の提案・成果物への愛着で判断が歪んでいないか？ | 感情ヒューリスティック | |
| 3 | 反対意見を積極的に求めたか？ | 集団思考 | |
| 4 | 過去の成功事例への類推に引きずられていないか？ | 顕著性バイアス | |
| 5 | 信頼できる代替案を具体的に検討したか？ | 確認バイアス | |
| 6 | 1年後に再判断するなら何の情報が欲しいか？ | 利用可能性バイアス | |
| 7 | 判断の根拠となる数値の出所を把握しているか？ | アンカリング | |
| 8 | 計画は楽観的すぎないか？ | 楽観バイアス | |
| 9 | 最悪ケースは十分に悪いか？ | 災害無視 | |
| 10 | 過度に慎重になっていないか？ | 損失回避 | |
| 11 | 一部の良い結果に引きずられて全体を過大評価していないか？ | ハロー効果 | |
| 12 | 過去の判断（ADR、計画、発言）にアンカーされていないか？ | サンクコスト | |

### 判定基準

| ⚠️ + ❌ の合計 | アクション |
|---------------|----------|
| 0-3 | 判断を実行してよい |
| 4-6 | 最も深刻な⚠️/❌に対して具体的対策を設計してから実行 |
| 7以上 | **判断を保留**。追加分析を行い、再度監査する |
